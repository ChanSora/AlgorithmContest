# DP题解

## P1880石子合并 区间DP板子题

n堆石子围成一个圈，每堆石子数目不同， 要进行合并，每次合并给答案提供两堆石子数量和的贡献，求最小贡献值和最大贡献值。

### DP过程

按段落长度枚举进行DP， 每次合并两个小段，再加上合并时的贡献，求最值即可。

$设A_{i, j}表示:区间[i, j]上石子合并的最大/最小贡献值.$

$$
A_{i, j} = max/min(A_{i, k} + A_{k + 1, j} + S_{i, j})\quad(i \leq k \leq j)
$$

### 几个需要注意的点

#### 1.破环成链

复制一份到末尾然后考虑2n长度段上n长度段的最值。

注意memcpy目标指针写在第一个参数位置

#### 2.初始化

实现的时候我们用两个数组分别储存最大最小值，最大值数组初始化为0，最小值数组初始化为inf。但是要注意DP开始时最小值数组长度为1的段要初始化为0（相当于长度为0的段提供了0贡献）。

## P4342Polygon 区间DP变式

在P1880的基础上，边可以有两种模式：把两端相乘和相加。，同时顶点上的初始值可以为负值。不用累计先前贡献。

### DP过程

这题有个思维陷阱：误以为只需要根据边的模式，每次求最大值就行了。但是由于顶点值可以为负数，所以可能会出现两个负数相乘得到最大值的情况。因此我们还需要存储最小值。

$设A_{i,j}为[i,j]上的最大值， B_{i,j}为最小值。$

$$
A_{i, j} = max(A_{i, k} (+/*) A_{k + 1, j}, B_{i, k} (+/*) B_{k + 1, j})\quad(i \leq k \leq j)
$$  

## CF149D 括号匹配

题目给定了一个合法的括号序列(每对括号都能正确匹配，不会出现有一个括号落单的情况)，现在我们要对括号进行染色，染色有三条规则：染色可以染成红蓝两色，或者不染色；每对括号要么两个都不染色，要么有且仅有一个括号被染色；相邻的括号颜色不能相同，但可以都不染色。问题是：统计所有合法的染色方案数，对1e9 + 7取模。

### DP过程

很容易观察到括号序列的一个嵌套递归性质。作为DP练习，我们也可以对它进行DP处理：

首先考虑最小的括号序列：()，它有5种染色情况。

接着，在括号序列逐渐扩大的过程中，我们需要分两种情况讨论:

1. 在原来的括号外面加了一层括号:如(())

2. 多个括号组合了起来:如()()

对第一种情况，我们需要考虑两边的括号是否匹配，同时新加的括号必须有合法性；

对第二种情况，我们只需要考虑拼合的时候中间两个括号能否满足规则即可。

实现方法：

括号匹配：使用栈来维护括号序列，"("括号遇到")"括号就弹出，再给每个括号带一个位置的tag，这样就可以确定每个括号是跟谁匹配了；

括号匹配好之后，我们维护一个括号值的前缀和，来确定一段区间上的括号是否匹配：如果"("是1,")"是-1,如果中间全部匹配，那么中间一段的前缀和是0；接着，就可以用滑动窗口法，按2递增处理所有序列了。

这里我们可以把状态压缩成一个数来枚举情况，不然每次枚举太麻烦了；对所有情况模拟即可。注意不要漏判断条件了。

对第一种情况，合法的外层括号直接继承小括号的方案数；

对第二种情况，我们只需要将括号划分为第一组和第二组。不需要每一组都算，因为方案数会记重。方法数就是两小块方法数的乘积。

### 注意事项

不要记重，这在DP过程中已经说过了；

注意取模运算。不是(ll)(a \* b) % mod, 而是(ll)a \* b % mod!

## UVa12991 Game Rooms

一栋n层高的酒店，每层都住着喜欢打乒乓球跟喜欢游泳的顾客，现要在每一层楼修一个乒乓球室或者是游泳池以满足顾客需求（必须修一个），依据类型，顾客会前往最近一层楼的娱乐设施。现在，如果记每位顾客的移动距离为楼层数，求如何规划修建这些娱乐设施使所有顾客走的总距离最小？

题目可以等价为：一段序列，每点上有两个参数，现在按黑白两色给序列分段染色，黑白分别对应两个参数，同色的参数不用计算，异色的贡献计算方法为：就近走到同色位置的距离长 * 自身的参数值，问如何染色使总贡献最小？

### DP 过程

个人认为这是一道线性DP题（为什么被放在区间DP里面了？）

既然是分段染色，那么很自然的一个想法就是按段转移（线性DP的经典思路）：

设$dp_{k, 0/1}$表示从1到k，最后一段是黑/白的最小贡献值。那么有

$$dp_{k,0/1} = min_{0 \leq j < k}(dp_{k, 0/1}, dp_{j, 1/0} + cost(j + 1, k, 0/1))$$

这里cost(i, j, 0/1) 表示将[i, j]染成黑白所要付出的代价。

现在我们只需考虑cost函数的算法即可。

异色会前往最近的同色，因此我们只需要在中间一分为二，左边的往左走，右边的往右走就可以了。

如何O(1)的计算每次的贡献？我们采用二阶前缀和，即对前缀和再做一次前缀和，同时做后缀和和二阶后缀和。

这样，随着离边界的距离越远，远处的参数值就会线性的叠加，完成了我们需要的乘以距离长的操作。

### 注意事项

1. 本题数据取值范围较大，注意开long long。

2. 多测试点注意每次重初始化的范围

3. 使用0/1来状压，在实际操作过程中容易混淆，注意定义清楚每个参数所代表的意义，如cost的第三个参数在写的时候就出了bug：把一段区间染成0的贡献值并不要用0的总距离，而是应该是用1的总距离来算。（0呆在原地不动就可以了，而1需要前前后后移动）

4. 注意代码小细节，不要写错数组了！(ppre和pre,手一滑就会打多)
